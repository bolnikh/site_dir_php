# Шаг 20: Функциональные тесты — спецификация

## Цель

Создать исчерпывающий набор автоматизированных функциональных тестов на Playwright, покрывающий все пользовательские сценарии и страницы проекта «Каталог сайтов».

## Инструменты

- **Playwright** (Node.js) — end-to-end тестирование в реальном браузере (Chromium)
- **PHPUnit** (опционально) — для модульных тестов PHP-хелперов

---

## Категория 1: Дымовые тесты (Smoke)

### Цель
Убедиться, что все страницы открываются без ошибок (HTTP 200, нет PHP-ошибок, нет 500).

### Тестовые данные
Требуется минимальный набор: хотя бы 1 корневой раздел, хотя бы 1 опубликованный сайт.

### Сценарии

| # | Тест | URL | Ожидаемый результат |
|---|---|---|---|
| SMK-01 | Главная | `/` | 200, заголовок H1 «Каталог сайтов» |
| SMK-02 | Раздел | `/section/{slug}` | 200, название раздела в H1 |
| SMK-03 | Подраздел | `/section/{child-slug}` | 200, хлебные крошки глубиной ≥2 |
| SMK-04 | Сайт | `/site/{slug}` | 200, название сайта в H1 |
| SMK-05 | О нас | `/about` | 200, заголовок «О нас» |
| SMK-06 | Правила | `/rules` | 200, заголовок «Правила» |
| SMK-07 | Добавить сайт | `/add` | 200, форма с полями |
| SMK-08 | Свяжитесь с нами | `/contact_us` | 200, форма обратной связи |
| SMK-09 | Логин модератора | `/moderator/login` | 200, форма логина |
| SMK-10 | 404 — раздел | `/section/nonexistent` | 404 |
| SMK-11 | 404 — сайт | `/site/nonexistent` | 404 |
| SMK-12 | 404 — маршрут | `/nonexistent-page` | 404 |

### Ассерты (для каждого)
```typescript
await expect(page).toHaveTitle(/Каталог сайтов/);
await expect(page.locator('body')).not.toContainText('Fatal error');
await expect(page.locator('body')).not.toContainText('Warning:');
await expect(page.locator('body')).not.toContainText('Parse error');
await expect(page.locator('body')).not.toContainText('Stack trace');
```

|---|---|---|

| BC-02 | Корневой раздел `/section/{slug}` | Начало > НазваниеРаздела |
| BC-03 | Подраздел `/section/{child-slug}` | Начало > Родитель > Подраздел |
| BC-04 | Подподраздел (3 уровня) | Начало > Родитель > Подраздел > Подподраздел |
| BC-05 | Сайт `/site/{slug}` | Начало > ... > РазделСайта > НазваниеСайта |
| BC-06 | О нас `/about` | Начало > О нас |
| BC-07 | Правила `/rules` | Начало > Правила |

### Ассерты
- Последний элемент крошек — текст без ссылки (текущая страница)
- Все предыдущие элементы — ссылки (`<a href="...">`)
- «Начало» всегда ссылка на `/`
- Крошки визуально присутствуют в `<nav aria-label="breadcrumb">`

## Категория 7: Форма добавления сайта

### Общий подход
Для всех форм: сначала тестируем **неверные** заполнения и проверяем ошибки, затем **верное** заполнение и проверяем успех.

### 7-A. JS-валидация (клиентская)

| # | Тест | Действие | Ожидаемая ошибка |
|---|---|---|---|
| ADD-JS-01 | Пустой раздел | Не выбрать раздел, заполнить остальное верно | «Выберите раздел» |
| ADD-JS-02 | Пустое название | Оставить название пустым | «Введите название сайта» |
| ADD-JS-03 | Длинное название | 513+ символов | «Название не должно превышать 512 символов» |
| ADD-JS-04 | Пустой URL | Оставить URL пустым | «Введите URL сайта» |
| ADD-JS-05 | Некорректный URL | `not-a-url` | «Введите корректный URL» |
| ADD-JS-06 | URL без протокола | `example.com` | «Введите корректный URL» |
| ADD-JS-07 | Пустое описание | Оставить описание пустым | «Введите описание сайта» |
| ADD-JS-08 | Некорректный email | `not-an-email` | «Введите корректный email» |
| ADD-JS-09 | Не приняты правила | Чекбокс не отмечен | «Необходимо принять условия» |
| ADD-JS-10 | Все поля пустые | Сабмит без заполнения | Ошибка на первом поле, скролл к нему |
| ADD-JS-11 | Ошибки исчезают | Вызвать ошибку, потом исправить | Класс `is-invalid` убран |

## Категория 14: Сквозной сценарий (End-to-End)

### Сценарий: пользователь добавляет сайт → модератор одобряет → сайт виден всем

1. **Гость** заходит на `/add`
2. Заполняет форму валидными данными
3. Отправляет → flash «Сайт отправлен на модерацию»
4. **Модератор** заходит на `/moderator/login`, входит
5. Видит новый сайт в списке на модерации
6. Открывает сайт, нажимает «Одобрить»
7. **Гость** заходит на главную `/` → сайт в списке последних
8. Заходит в раздел сайта → сайт в списке
9. Открывает страницу сайта → данные корректны

### Сценарий: пользователь добавляет сайт → модератор отклоняет → сайт не виден

1. **Гость** добавляет сайт
2. **Модератор** отклоняет сайт
3. Сайт отсутствует на главной, в разделе, недоступен по прямой ссылке

### Сценарий: модератор редактирует сайт перед публикацией

1. **Гость** добавляет сайт с названием «Тестовый сайт»
2. **Модератор** открывает сайт, меняет название на «Обновлённый сайт», описание на «Новое описание», меняет раздел
3. Нажимает «Сохранить и опубликовать»
4. На странице сайта — «Обновлённый сайт», «Новое описание», правильный раздел

## Категория 16: Кэширование (Redis)

### Примечание
Эти тесты требуют доступа к Redis для проверки ключей. Можно реализовать через отдельный эндпойнт `/debug/cache-stats` (только в dev-режиме) или через прямое подключение из тестового хука.

| # | Тест | Ожидаемый результат |
|---|---|---|
| CACHE-01 | Главная — sections:tree | Ключ `catalog:sections:tree` создаётся после захода на `/` |
| CACHE-02 | Последние сайты | Ключ `catalog:sites:recent:10` создаётся |

| CACHE-04 | Инвалидация при добавлении | После публикации сайта ключи сбрасываются |

## Приоритеты реализации

### Фаза 1 (обязательный минимум)
- SMK: дымовые тесты всех страниц
- BC: хлебные крошки

- HOME, SEC, SITE: проверка контента страниц
- STAT: статические страницы
- CNT: форма обратной связи
- SEC-CRUD: управление разделами
- PAG: пагинация

### Фаза 3 (безопасность и граничные случаи)
- XSS: XSS-защита
- ACC: контроль доступа
- CACHE: кэширование
- Граничные случаи всех форм
