# Шаг 9: Статические страницы «О нас» и «Правила»

## Цель

Реализовать страницы `/about` и `/rules` с фиксированным содержимым.

## Маршруты

- `/about` → `pages/about.php`
- `/rules` → `pages/rules.php`

## Страница «О нас» (`/about`)

### Хлебные крошки
```
Начало > О нас
```

### Содержание

Из `pages/about.txt`:
- Заголовок: ★ О НАС ★
- Текст о проекте HomeCatalog
- Разделы: «Зачем мы нужны?», «Как мы работаем?», «Для кого этот каталог?»
- Контакты: Email, ВКонтакте

Реализация: статический HTML в `pages/about.php`. Данные можно хранить в БД (отдельная таблица `pages`), но для простоты — прямо в шаблоне.

## Страница «Правила» (`/rules`)

### Хлебные крошки
```
Начало > Правила
```

### Содержание

Из `pages/rules.txt`:
- Заголовок: ПРАВИЛА КАТАЛОГА
- 5 разделов правил:
  1. Добавление сайтов
  2. Модерация
  3. Поведение на сайте
  4. Ответственность
  5. Изменение правил
- Контактный email

Реализация: статический HTML в `pages/rules.php`.

## Альтернативный подход (если нужно редактирование)

Создать таблицу `pages`:
```sql
CREATE TABLE pages (
    id SERIAL PRIMARY KEY,
    slug VARCHAR(255) UNIQUE NOT NULL,
    title VARCHAR(512) NOT NULL,
    content TEXT DEFAULT '',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

INSERT INTO pages (slug, title, content) VALUES
('about', 'О нас', '...'),
('rules', 'Правила', '...');
```

Тогда страницы загружаются из БД, а модератор может их редактировать. Но в текущих требованиях этого нет — реализуем статически.
