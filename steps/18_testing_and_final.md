# Шаг 18: Тестирование и финальная проверка

## Цель

Проверить работоспособность всех компонентов сайта перед сдачей.

## Чек-лист функционального тестирования

### Публичная часть

- [ ] **Главная страница (`/`)**
  - [ ] Сетка разделов отображается (5 корневых разделов)
  - [ ] Каждый раздел — ссылка на `/section/{slug}`
  - [ ] Подразделы перечислены под названиями
  - [ ] Кнопка «Добавить сайт» отображается
  - [ ] Список последних 10 сайтов (если есть опубликованные)
  - [ ] Нет хлебных крошек

- [ ] **Страница раздела (`/section/{slug}`)**
  - [ ] Хлебные крошки: Начало > Раздел (или глубже)
  - [ ] Название и описание раздела
  - [ ] Подразделы (если есть)
  - [ ] Список сайтов с пагинацией
  - [ ] Пагинация работает корректно
  - [ ] Сайты из дочерних разделов включаются

- [ ] **Страница сайта (`/site/{slug}`)**
  - [ ] Хлебные крошки полные
  - [ ] Название сайта (H1)
  - [ ] Дата добавления
  - [ ] Описание сайта
  - [ ] Неопубликованные сайты недоступны

- [ ] **О нас (`/about`)**
  - [ ] Контент соответствует макету
  - [ ] Хлебные крошки: Начало > О нас

- [ ] **Правила (`/rules`)**
  - [ ] Контент соответствует макету
  - [ ] Хлебные крошки: Начало > Правила

- [ ] **Добавить сайт (`/add`)**
  - [ ] Форма отображается
  - [ ] Селект разделов показывает дерево
  - [ ] Summernote работает для описания
  - [ ] JS-валидация проверяет обязательные поля
  - [ ] Серверная валидация возвращает ошибки
  - [ ] Успешная отправка → flash + редирект
  - [ ] Сайт попадает в БД со статусом 0
  - [ ] Slug генерируется из названия

### Модераторская часть

- [ ] **Логин (`/moderator/login`)**
  - [ ] Форма входа
  - [ ] Правильный логин/пароль → редирект на список
  - [ ] Неправильный логин/пароль → ошибка

- [ ] **Защита страниц**
  - [ ] `/moderator/list` без авторизации → редирект на логин
  - [ ] `/moderator/sections` без авторизации → редирект на логин

- [ ] **Список на модерацию (`/moderator/list`)**
  - [ ] Сайты со статусом 0 отображаются
  - [ ] Поиск по названию работает
  - [ ] Пагинация работает
  - [ ] Клик по сайту → `/moderator/moderate?id=`

- [ ] **Модерация сайта (`/moderator/moderate?id=`)**
  - [ ] Все данные сайта отображаются
  - [ ] Кнопка «Одобрить» работает
  - [ ] Кнопка «Отклонить» работает
  - [ ] Форма редактирования + «Сохранить и опубликовать» работает
  - [ ] Email отправляется (если указан)
  - [ ] Flash-сообщения после действий

- [ ] **Управление разделами (`/moderator/sections`)**
  - [ ] Дерево разделов отображается
  - [ ] Фильтр по названию работает
  - [ ] Кнопка «Добавить подраздел» работает
  - [ ] Кнопка «Редактировать» работает
  - [ ] Кнопка «Удалить» работает (только для пустых разделов)
  - [ ] Нельзя удалить раздел с сайтами

- [ ] **Добавление раздела (`/moderator/sections/add`)**
  - [ ] Форма с выбором родителя
  - [ ] Slug автогенерируется
  - [ ] Сохранение работает, path вычисляется

- [ ] **Редактирование раздела (`/moderator/sections/edit?id=`)**
  - [ ] Поля предзаполнены
  - [ ] Изменение родителя пересчитывает path
  - [ ] Блок «Опасная зона» показывает статистику

## Технические проверки

- [ ] **XSS-защита**
  - [ ] Все пользовательские данные экранируются через `h()`
  - [ ] HTML из редактора фильтруется через HTMLPurifier
  - [ ] Попытка XSS в названии сайта → безвредна

- [ ] **SQL-инъекции**
  - [ ] Все запросы через PDO prepared statements
  - [ ] Нет конкатенации пользовательских данных в SQL

- [ ] **Кэширование**
  - [ ] Redis-ключи создаются с префиксом `catalog:`
  - [ ] TTL соблюдаются
  - [ ] Инвалидация работает при изменениях
  - [ ] При недоступности Redis сайт не падает

- [ ] **Email**
  - [ ] В debug-режиме используется `mail()`
  - [ ] В production — SMTP через PHPMailer
  - [ ] Письма уходят в UTF-8

- [ ] **Пагинация**
  - [ ] По 20 сайтов на страницу (настраивается)
  - [ ] Корректное количество страниц
  - [ ] Нет ошибок при page=0, page=-1, page=999999

- [ ] **404**
  - [ ] Несуществующий раздел → 404
  - [ ] Несуществующий сайт → 404
  - [ ] Несуществующий маршрут → 404

- [ ] **Flash-сообщения**
  - [ ] Появляются после редиректа
  - [ ] Исчезают после обновления
  - [ ] Правильные типы (success, error, warning, info)

## Быстрые автотесты (опционально)

- Запустить `composer validate` — валидация composer.json
- PHPStan / Psalm на уровне 1 — базовый статический анализ
- Проверка синтаксиса всех PHP-файлов: `find . -name "*.php" -exec php -l {} \;`
