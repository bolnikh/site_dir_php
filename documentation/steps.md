Отличный проект для начала! Вот подробный роадмап по вайбкодингу для **Каталога сайтов**. Разобьем на небольшие итерации — каждую можно сделать за 1-2 часа.

---

## 🗺️ Роадмап (10 итераций)

### **Итерация 1: Базовая структура + Docker**
**Цель:** Запустить окружение и увидеть "Hello World"

- [ ] Создать `docker-compose.yml` (php-fpm, nginx, postgres)
- [ ] Создать `Dockerfile` для PHP с расширениями (pdo_pgsql)
- [ ] Настроить nginx virtual host (корень `/public`)
- [ ] Создать `index.php` в `/public` — просто `phpinfo()` или "Hello World"
- [ ] Запустить и убедиться что открывается в браузере

**Проверка:** Открываете `http://localhost` → видите "Hello World"

---

### **Итерация 2: Шаблоны + Базовая структура страниц**
**Цель:** Сделать общий layout и все страницы-заглушки

- [ ] Создать `config/config.php` (пока только `SITE_NAME`, `ITEMS_PER_PAGE = 20`)
- [ ] Создать `public/css/style.css` (подключить Bootstrap через CDN)
- [ ] Сделать `templates/layout.php` (верхняя плашка с названием и кнопкой "Добавить сайт", подвал)
- [ ] Создать страницы-заглушки:
  - `public/index.php` (главная)
  - `public/section.php` (страница раздела)
  - `public/site.php` (страница сайта)
  - `public/about.php`
  - `public/rules.php`
  - `public/add.php`
- [ ] В каждой странице подключить `layout.php`

**Проверка:** Переход по `/index.php`, `/about.php` и т.д. — видите одинаковую шапку/подвал

---

### **Итерация 3: База данных и модели (PDO)**
**Цель:** Создать таблицы и подключение к БД

- [ ] Создать `config/db.php` (подключение PDO к Postgres)
- [ ] Написать SQL миграции (вручную выполнить через `psql`):
  - `sections` (id, parent_id, path, name, slug, description, created_at, updated_at) — см. @documentation/db_schema.md
  - `sites` (id, section_id, name, slug, url, description, email, status, created_at, moderated_at) — см. @documentation/db_schema.md
  - `users` (id, username, password, active, created_at, updated_at) — см. @documentation/db_schema.md
- [ ] Создать `models/Section.php` (CRUD + getTree)
- [ ] Создать `models/Site.php` (CRUD + getBySection + getRecent)

**Проверка:** Написать тестовый скрипт, который вставляет раздел и сайт в БД

---

### **Итерация 4: Главная страница**
**Цель:** Главная показывает разделы и последние 10 сайтов

- [ ] В `index.php`:
  - Загрузить из БД все разделы (построить дерево для вывода в несколько столбцов)
  - Загрузить 10 последних сайтов (status = 'published')
- [ ] Вывести разделы в виде сетки (Bootstrap row/col)
- [ ] Вывести список последних сайтов (название + ссылка)
- [ ] Кнопка "Добавить сайт" ведет на `/add.php`

**Проверка:** На главной видите разделы и список сайтов (пока тестовых)

---

### **Итерация 5: Страница раздела + пагинация**
**Цель:** Показывать сайты в разделе с постраничной навигацией

- [ ] В `section.php` через GET-параметр `slug` определить текущий раздел
- [ ] Построить хлебные крошки (рекурсивно подняться по parent_id)
- [ ] Показать подразделы (если есть)
- [ ] Показать сайты текущего раздела с пагинацией:
  - `LIMIT $offset, $per_page`
  - Количество страниц = ceil(total / per_page)
- [ ] Сделать навигацию (Previous / Next / номера страниц)

**Проверка:** Переход по `/section.php?slug=dom` → видите подразделы и сайты (пагинация работает)

---

### **Итерация 6: Страница отдельного сайта**
**Цель:** Полная страница с информацией о сайте

- [ ] В `site.php` через `?id=` загрузить сайт (только published)
- [ ] Построить хлебные крошки (раздел → подраздел → название сайта)
- [ ] Вывести:
  - Название сайта (заголовок)
  - Дата добавления
  - Описание (htmlspecialchars)
  - Ссылка на сайт (кнопка "Перейти")
- [ ] Если сайт не найден — 404

**Проверка:** Переход на страницу тестового сайта → видите все поля

---

### **Итерация 7: Форма добавления сайта (frontend)**
**Цель:** Пользователь может отправить сайт на модерацию

- [ ] В `add.php` сделать форму:
  - Название (text, required)
  - URL (url, required)
  - Описание (textarea, можно подключить простой редактор — например, SimpleMDE)
  - Email (email, опционально)
  - Checkbox "Согласен с правилами" (required)
- [ ] Валидация на JS (подсветка ошибок)
- [ ] Отправка на сервер (POST в `handlers/add_site.php`)
- [ ] Сохранение в БД со статусом `'pending'`
- [ ] После сохранения — редирект на главную с сообщением "Сайт отправлен на модерацию"

**Проверка:** Заполнить форму → сайт появляется в БД со статусом `pending`

---

### **Итерация 8: Модерация (авторизация + список на модерацию)**
**Цель:** Модератор может залогиниться и увидеть список сайтов

- [ ] Создать `config/moderator.php` (логин + хеш пароля)
- [ ] Создать `admin/login.php` (форма логина, сессия)
- [ ] Создать `admin/index.php` (список сайтов со статусом `pending`, пагинация)
  - Макет: @documentation/pages/moderator_list.txt
- [ ] В списке: название сайта (ссылка на страницу модерации), раздел, дата
- [ ] Сделать logout

**Проверка:** Залогиниться → увидеть список ожидающих модерации сайтов

---

### **Итерация 9: Модерация (редактирование + одобрение/отклонение)**
**Цель:** Модератор может управлять сайтом

- [ ] Создать `admin/moderate.php?id=` (страница модерации)
- [ ] Вывести текущие данные сайта
- [ ] Кнопки:
  - **Одобрить** → статус `'published'`, если есть email → отправить письмо
  - **Отклонить** → статус `'rejected'`, если есть email → отправить письмо
- [ ] Форма редактирования:
  - Селект раздела (все разделы деревом)
  - Название (текст)
  - Описание (textarea)
  - Кнопка "Сохранить и опубликовать"
- [ ] Написать функцию отправки письма — см. @documentation/technical.md (SMTP + шаблоны)

**Проверка:** Одобрить сайт → он появляется на главной и в разделе

---

### **Итерация 10: Доработки и полировка**
**Цель:** Сделать проект законченным

- [ ] Добавить страницу "О нас" и "Правила" (текст из конфига или БД)
- [ ] Добавить SEO-дружественные ЧПУ-ссылки — см. @documentation/technical.md (nginx config)
- [ ] Добавить валидацию на сервере — см. @documentation/technical.md (правила валидации, XSS защита)
- [ ] Сделать красивый вывод ошибок — см. @documentation/technical.md (flash-сообщения через сессию)
- [ ] Подключить визуальный редактор — см. @documentation/technical.md (TinyMCE/Summernote)
- [ ] Настроить Redis для кэша — см. @documentation/technical.md (ключи, TTL)
- [ ] Добавить CRUD управления разделами:
  - `admin/sections.php` — дерево разделов, см. @documentation/pages/sections_tree.txt
  - `admin/section_add.php` — добавить раздел, см. @documentation/pages/section_add.txt
  - `admin/section_edit.php?id=` — редактировать/удалить раздел, см. @documentation/pages/section_edit.txt
- [ ] Написать простой `README.md` по установке
- [ ] Проверить адаптивность (Bootstrap делает 90% работы)

**Проверка:** Весь функционал работает, ссылки красивые, ошибки обрабатываются

## 💡 Советы для вайбкодинга

1. **Каждую итерацию делайте работающей** — не переходите дальше, пока текущая не работает
2. **Коммитьте после каждой итерации** (можно локально, но лучше на гит)
3. **Claude + Deepseek используйте так:**
   - Claude для архитектуры, сложных SQL запросов, nested set дерева
   - Deepseek для быстрых однотипных задач (например, "сделай CRUD для модели Section")
4. **Не бойтесь переписывать** — если что-то пошло не так, проще переделать маленькую итерацию
