## Как запустить

### 1. Запустить тестовую среду

первый раз

```
sudo docker compose -f docker-compose.test.yml up --build --abort-on-container-exit
```

потом

```
sudo docker compose -f docker-compose.test.yml up -d
```

миграции для бд для тестовой сборки tests/migration

для очистки бд 

```
sudo docker compose -f docker-compose.test.yml down -v
```

### 2. Запустить все тесты

```
  cd tests && npx playwright test
```

### 3. Конкретная группа

```
  npx playwright test public/smoke.spec.ts
  npx playwright test moderator/auth.spec.ts
```

### 4. С HTML-отчётом

```
  npx playwright test --reporter=html
  npx playwright show-report
```

### 5. В UI-режиме (отладка)

```
  npx playwright test --ui
```

### 6. Запуск тестов для виртуальной машины

Лучше всего запустить на localhost пробросив порт 8080 на хост машину из виртуальной машины.

Если так не получилось - проект не запускается на localhost, а из под IP 192.168.50.20 . 

```
BASE_URL=http://192.168.50.20:8080 npx playwright test
```

для запуска одного теста

```
BASE_URL=http://192.168.50.20:8080 npx playwright test test_name
```

под виндоус

```
$env:BASE_URL="http://192.168.50.20:8080"; npx playwright test
```

##   docker-compose.test.yml

  Тестовое окружение, полностью изолированное от основного:

  ┌────────────┬────────────────────────────┬─────────────────────────────────────────────────────┐
  │   Сервис   │            Порт            │                     Особенность                     │
  ├────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ nginx      │ 8080                       │ healthcheck через php                               │
  ├────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ php        │ —                          │ SMTP направлен в MailHog                            │
  ├────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ postgres   │ 5433                       │ загружает tests/fixtures/db-seed.sql после миграций │
  ├────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ redis      │ 6380                       │ —                                                   │
  ├────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ mailhog    │ 1025 (SMTP), 8025 (Web UI) │ перехватывает все письма                            │
  ├────────────┼────────────────────────────┼─────────────────────────────────────────────────────┤
  │ playwright │ —                          │ ждёт готовности nginx, затем прогоняет тесты        │
  └────────────┴────────────────────────────┴─────────────────────────────────────────────────────┘

  tests/Dockerfile

  На базе официального mcr.microsoft.com/playwright:latest, устанавливает только npm-зависимости.

  Запуск

### Запустить тесты
  docker compose -f docker-compose.test.yml up --build --abort-on-container-exit

### Остановить и очистить
  docker compose -f docker-compose.test.yml down -v

  Порты сдвинуты относительно продакшена (8080, 5433, 6380), чтобы основное приложение и тесты могли работать
  одновременно без конфликтов. Почта в тестах уходит в MailHog (веб-интерфейс на http://localhost:8025).
